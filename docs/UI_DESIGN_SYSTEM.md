# 数学 AI 复习助手 UI 设计系统

## 设计目标

界面面向高校学生的高频复习场景，强调清晰、可信、低干扰和可连续操作。视觉以蓝白教育产品为主，状态色只表达明确含义，不使用装饰性渐变、夸张圆角或虚假数据。

## 页面结构

### 学生端

- 桌面：232px 左侧导航、64px 顶部工具栏、居中内容区。
- 侧栏可折叠到 76px，当前路由使用主色高亮。
- 手机：顶部保留搜索与菜单入口，底部提供 5 个真实功能导航。
- 内容在 760px 以下切换为单列，触控操作最小高度 44px。

### 管理端

- 使用与学生端一致的浅色侧栏和顶部栏。
- 页面头、筛选区、表格、状态标签和弹窗遵循统一视觉。
- 管理首页只显示 `/api/admin/stats` 返回的真实统计。

## 数学内容

`FormulaContent` 是数据库内容和 AI 内容唯一允许使用 `v-html` 的组件。处理顺序：

1. 暂存公式文本。
2. 对普通文本执行 HTML 转义。
3. 只转换受控的标题、加粗、代码和换行。
4. 使用 KaTeX 渲染公式，`trust: false`、`throwOnError: false`。
5. 块级公式允许横向滚动，避免手机端溢出。

禁止页面直接对题干、答案、解析和 AI 文本使用 `v-html`。

## 交互规范

- 题目详情首次加载不请求答案；点击“查看答案与解析”后才调用受保护接口。
- 主要命令使用主按钮，次要命令使用普通或文字按钮。
- 删除、禁用等高风险操作必须二次确认。
- 无真实数据时显示空状态，不生成演示数字或伪趋势。
- Loading 使用 Skeleton 或 Element Plus loading，错误由统一请求拦截器提示。

## 可访问性与响应式

- 图标按钮提供 `aria-label` 或 tooltip。
- 页面标题保持单一 H1，模块标题使用 H2/H3。
- 键盘焦点使用主色外轮廓。
- 支持 `prefers-reduced-motion`。
- 手机端不依赖 hover，固定底栏之外保留足够底部空间。
